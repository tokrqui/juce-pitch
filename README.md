# JucePitch — краткий обзор и инструкции по сборке

Это репозиторий плагина на базе JUCE (питч-шейфтер). Ниже — краткий разбор проекта, найденные зависимости и шаги для сборки на Windows.

Ключевые моменты проекта
- Тип: аудиоплагин (JUCE) — генерируется VST3-артефакт.
- Сборка: CMake (требуется CMake >= 3.22) с пресетом для Visual Studio 2022 (см. `CMakePresets.json`).
- Основные исходники:
  - `Source/PluginProcessor.*` — процессинг и состояние плагина
  - `Source/PluginEditor.*` — GUI обёртка плагина
  - `dsp/PitchShifterDSP.*` — DSP-ядро питч-шифтера
  - `gui/*` — набор виджетов и визуальной логики

Что я проверил
- Прочитал `CMakeLists.txt` и `CMakePresets.json`.
- Просканировал репозиторий на наличие `.cpp`/`.h` (основные исходники присутствуют).
- Непосредно в репозитории не найден каталог `vst3sdk` и не обнаружен явный каталог `../JUCE` рядом с проектом (CMake ожидает наличие JUCE в `../JUCE`).

Проблемы и рекомендации
- Зависимости:
  - `CMakeLists.txt` содержит строку `add_subdirectory(../JUCE ${CMAKE_BINARY_DIR}/JUCE)` — это означает, что сборка ожидает локальную копию JUCE в соседней директории (`../JUCE`) relative к корню проекта. Убедитесь, что у вас есть клон JUCE (или подключён как submodule) в этой позиции, либо поправьте путь.
  - Путь к VST3 SDK устанавливается через `juce_set_vst3_sdk_path(${CMAKE_CURRENT_SOURCE_DIR}/../vst3sdk)`. В репозитории не найден `vst3sdk`. Для сборки VST3 нужен VST3 SDK (Steinberg) — добавьте его в `../vst3sdk` или настройте путь в CMake.
- Названия папок: в CMake используются `source/` (строчные), а в дереве файлов папка — `Source/` (с заглавной). На Windows это не мешает (case-insensitive), но может создать проблемы на CI/Linux. Рекомендую привести к единообразию (например, переименовать в `source/`) и обновить CMake.
- CMakePresets настроены на цель `JucePitch_VST3` — это нормальное имя для VST3-плагина, но сборка упадёт, если VST3 SDK отсутствует.

Быстрые шаги для сборки на Windows (Visual Studio 2022):
1. Установите зависимости:
   - CMake >= 3.22
   - Visual Studio 2022 с компонентами C++ Desktop и поддержкой CMake
2. Получите JUCE рядом с проектом (в одну папку выше), например:
   - git clone --depth 1 https://github.com/juce-framework/JUCE.git ../JUCE
   либо: добавьте JUCE как git submodule и разместите его в `../JUCE`.
3. Получите VST3 SDK рядом с проектом:
   - git clone --depth 1 https://github.com/steinbergmedia/vst3sdk.git ../vst3sdk
4. Откройте проект через CMakePresets (рекомендую использовать GUI VS):
   - В Visual Studio: File → Open → CMake...
   - Или в терминале: `cmake --preset windows-vs2022` и затем `cmake --build --preset windows-vs2022-debug`.
